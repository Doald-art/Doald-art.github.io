Slam 简述

  1. 概念：实时定位与地图的构建，或并发建图与定位
  2. SLAM问题可以描述为: 机器人在未知环境中从一个未知位置开始移动,在移动过程中根据位置估计和地图进行自身定位，同时在自身定位的基础上建造增量式地图，以绘制出外部环境的完全地图。
  3. 注意：slam 虽然是机器人导航的重要技术之一，但是二者并不等价，也就是说，slam 只是实现地图构建和即时定位

Slam 建图

  1. Gmapping 简介
    1. 概念与作用：gmapping 是 ros 中较为成熟的 slam 算法之一，gampping 可以根据移动机器人里程计数据和激光雷达数据二维的栅格地图。
    2. 安装指令：sudo apt install ros-noetic-gmapping
  2. Gmapping 节点说明

    1. 订阅的 topic

      • Tf: 用于雷达、底盘与里程计之间的坐标变换消息
      • Scan: slam 所需的雷达信息

    2. 发布的 topic

      • Map_metadata: 地图元数据
      • Map (nav_msgs/OccupancyGrid): 地图栅格数据，一般会在 rviz 中以图形化的方式显示
      • ~entropy (std_msgs/Float64): 机器人姿态分布熵估计 (值越大，不确定性越大)

    3. 服务：

      • Dynamic_map (nav_msgs/GetMap): 用于获取地图数据

    4. 参数：
      ~base_frame(string, default:”base_link”)

      • 机器人基坐标系。

      ~map_frame(string, default:”map”)

      • 地图坐标系。

      ~odom_frame(string, default:”odom”)

      • 里程计坐标系。

      ~map_update_interval(float, default: 5.0)

      • 地图更新频率，根据指定的值设计更新间隔。

      ~maxUrange(float, default: 80.0)

      • 激光探测的最大可用范围(超出此阈值，被截断)。

      ~maxRange(float)

      • 激光探测的最大范围。

    5. 所需的坐标变换

      1. 雷达坐标系–>基坐标系
        • 一般由 robot_state_publisher 或 static_transform_publisher 发布
      2. 基坐标系–>里程计坐标系
        • 一般由里程计节点发布
  3. Gmapping 使用
    1. 编写 gmapping 节点相关 launch 文件
  4. 执行：
    1. 先启动 Gazebo 仿真环境
    2. 启动地图绘制的 launch 文件
    3. 启动键盘控制节点，用于控制机器人运动建图

豆包给的解决方法

从提供的TF树图可知，map - odom 与 base_link - child_link 之间缺乏连接，导致无法建立 child_link 到 map 的变换。以下是针对性解决步骤：

1. 补充 odom 到 base_link 的变换 SLAM建图时，通常需通过里程计或SLAM算法发布 odom 到 base_link 的变换。若缺失，需检查SLAM节点配置：
  • 以Gmapping为例：确保启动文件中正确配置了坐标系关系，例如在启动参数中明确 odom_frame 和 base_frame： 若使用自定义SLAM方案，需确保代码中通过 tf2_ros 发布 odom 到 base_link 的实时变换。
2. 检查机器人状态发布 robot_state_publisher 需依赖关节状态构建TF树。确保：
  • 关节状态正常发布：运行 rostopic echo /joint_states，检查是否有机器人关节角度数据。若缺失，需启动 joint_state_publisher 或硬件驱动节点。
  • URDF配置正确：确认URDF中连杆关系包含 base_link 到 child_link 的正确定义，且通过 robot_state_publisher 加载：

3. 手动验证TF变换使用 tf_echo 工具检查关键变换： - 验证 odom 到 base_link： bash rosrun tf tf_echo odom base_link - 验证 base_link 到 child_link： bash rosrun tf tf_echo base_link child_link 若某一步骤无输出，说明对应变换未发布，需返回检查发布节点。通过补充 odom 到 base_link 的变换，并确保机器人TF树完整，即可建立 child_link 到 map 的连通路径，解决Rviz的TF错误。
