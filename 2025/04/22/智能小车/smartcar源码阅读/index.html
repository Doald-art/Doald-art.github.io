Keyop 键盘控制

代码作用：基于 ROS（Robot Operating System）的键盘遥控脚本，用于控制基于 Ackermann 转向的机器人。Ackermann 转向是一种常见的车辆转向模型，适用于汽车等机器人。脚本通过监听键盘输入，实时调整机器人速度和转向角，并将控制命令发布到 ROS 主题上。

导入模块

  • 导入 ROS 和 Python 标准库模块：

    • ackermann_msgs.msg.AckermannDriveStamped：用于发布 Ackermann 驱动消息。
    • sys, select, termios, tty：处理键盘输入。

  • Self 参数的作用和含义
    在 Python 中，self 是类方法的第一个参数，表示类的实例本身。通过 self，类的方法可以访问和修改该实例的属性和调用其他方法。它是类中定义实例变量和方法的桥梁。

    在这段代码中，self 的用法和含义如下：

    1. 访问实例属性

    self 用于访问和修改类实例的属性。例如：

    - self.speed 和 self.steering_angle 是 AckermannDriveKeyop 类的实例属性，用于存储当前速度和转向角。
    - 通过 self，这些属性可以在类的其他方法中被访问和修改。

    2. 调用实例方法

    self 用于在类内部调用其他实例方法。例如：

    - self.print_state() 调用当前实例的 print_state 方法，用于打印当前状态。
    - self.key_loop() 调用当前实例的 key_loop 方法，进入键盘监听循环。

    3. 在方法间共享数据

    self 允许在类的不同方法之间共享数据。例如：

    - self.speed 是一个实例属性，在 key_loop 方法中被修改。
    - 其他方法（如 pub_callback）可以访问 self.speed，从而实现数据共享。

    4. 初始化实例属性

    在 __init__ 方法中，self 用于初始化实例属性。例如：

    - 这些属性在实例化时被设置，供类的其他方法使用。

    5. 绑定 ROS 发布者和定时器

    self 用于存储 ROS 发布者和定时器的实例。例如：

    - self.motors_pub 是一个 ROS 发布者，用于发布控制命令。
    - 通过 self，其他方法（如 pub_callback）可以使用该发布者。

    总结

    self 是类实例的引用，用于：

    1. 访问和修改实例属性。

    2. 调用实例方法。

    3. 在方法间共享数据。

    4. 初始化实例属性。

    5. 绑定外部资源（如 ROS 发布者）。

      在这段代码中，self 是 AckermannDriveKeyop 类的核心，用于管理实例的状态和行为。

  • 根据命令行参数设置最大速度和最大转向角，默认值为 100 和 22。
  • 设置发布的 ROS 主题名称，默认为 ackermann_cmd。
  • 定义速度和转向角的范围。
  • 根据最大速度和转向角调整按键绑定的值。
  • 初始化速度和转向角为 0。
  • 创建 ROS 发布者，用于发布 Ackermann 驱动消息。
  • 设置定时器，每秒 5 次调用 pub_callback 方法。
  • 打印初始状态并进入按键循环。

按键循环
